# Firebase Secret Remediation - 2026-04-22

This note records the remediation of the GitHub secret-scanning alerts raised against the initial Flutter repo bootstrap.

## Trigger

- GitHub secret scanning opened two public alerts for Google API keys committed in `lib/firebase_options.dart` during the initial bootstrap history.
- The alerts pointed to commit `ee846e45` and the Android and iOS `apiKey` fields generated by FlutterFire.

## Remediation Applied In Repo

- removed tracked `lib/firebase_options.dart` from the mobile repo
- switched `FirebaseRuntime` to initialize Firebase from the native Android and iOS config files already kept local and gitignored
- added `lib/firebase_options.dart` to `.gitignore` to prevent future accidental commits of generated Firebase API keys
- updated the mobile docs to state that Firebase mobile initialization now depends on local `google-services.json` and `GoogleService-Info.plist`
- bumped the mobile build metadata to `1.0.3+25` and appended the `VERSION` history

## Follow-Up Outside Git

- If the exposed Google API keys remain active in Firebase or Google Cloud, rotate or restrict them there even after removing them from the repository tree.
- Secret-scanning alerts should only be marked resolved after the exposure has been remediated at both the repository and provider levels.
